### âœ… 3.1.1 Synchronous vs Asynchronous Methods

Node.js gives you **both sync and async** versions of file operations.

#### âŒ Synchronous (Blocking)
```js

// Blocks the entire process until file is read
const data = fs.readFileSync('data.txt', 'utf8');
console.log(data);

// Writes immediately â€” blocks until done
fs.writeFileSync('log.txt', 'App started');
```
- âœ… Simple to understand
- âŒ **Never use in production servers** â€” it **blocks** the event loop!

#### âœ… Asynchronous (Non-blocking) â€“ Recommended
```js
const fs = require('fs');

// Reads file in background
fs.readFile('data.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Writes without blocking
fs.writeFile('log.txt', 'User logged in', (err) => {
  if (err) console.error('Write failed:', err);
  else console.log('Log saved!');
});
```
- âœ… Doesn't block â€” perfect for servers
- Uses **callbacks** (error-first pattern)

### âœ… 3.1.3 Working with Directories

```js
const fs = require('fs');

// Create directory
fs.mkdir('uploads', (err) => {
  if (err && err.code !== 'EEXIST') {
    console.error('Failed to create folder');
  }
});

// Read directory contents
fs.readdir('uploads', (err, files) => {
  if (err) return console.error(err);
  console.log('Files:', files);
});

// Remove directory (must be empty)
fs.rmdir('old-backup', (err) => {
  if (err) console.error('Delete failed');
});
```

> ğŸ›  Use Case:  
Auto-create `logs/`, `uploads/`, or `temp/` folders on app startup.

### âœ… 3.1.4 File Stats: `fs.stat()`

Check if a file is a file, directory, size, creation date, etc.

```js

  if (err) return console.error(err);

  console.log('Is file?', stats.isFile());        // true
  console.log('Is directory?', stats.isDirectory()); // false
  console.log('Size:', stats.size);               // 1024 bytes
  console.log('Created:', stats.birthtime);
});
```

> ğŸ›  Use Case:  
Validate uploads â€” reject if it's a folder or too large.

### âœ… 3.1.5 Streams: `fs.createReadStream()` & `fs.createWriteStream()`

For **large files**, donâ€™t load everything into memory â€” **stream it in chunks**.

#### Example: Copy a Large Video File
```js

const writeStream = fs.createWriteStream('copy.mp4');

// Pipe data chunk by chunk

readStream.on('end', () => console.log('Copy complete!'));
```

### âœ… 3.1.7 Error Handling in File Operations

Always handle errors â€” files may not exist, or permissions may be denied.

```js
fs.readFile('missing.txt', 'utf8', (err, data) => {
  if (err) {
    if (err.code === 'ENOENT') {
      console.error('File not found');
    } else if (err.code === 'EACCES') {
      console.error('Permission denied');
    } else {
      console.error('Unknown error:', err);
    }
    return;
  }
  console.log(data);
});
```
